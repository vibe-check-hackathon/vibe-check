# VibeCheck Demo — Meta Info

| Section | Details |
| --- | --- |
| One-line premise | A self-referential demo where VibeCheck interviews its own product thesis in order to showcase the core experience: evidence-based questioning, structured hypothesis testing, and transparent decision-making. |
| Goal | Make the audience feel that VibeCheck is more than a generic AI assistant, genuinely useful for high-stakes decisions, and capable of surfacing evidence, uncertainty, and contradictions rather than pretending certainty. |
| Core message | VibeCheck is not just a chatbot that gives opinions. It is a decision system that asks better questions, tracks evidence, distinguishes fact from inference, and turns findings into an inspectable recommendation and term sheet. |
| Demo angle | The demo uses a light meta-joke: the system is asked to “vibe check” the concept itself, the AI interviewer tests the product thesis the way it would test a founder, and the audience sees the system’s own logic applied to its own pitch. |
| Recommended length | 2 to 3 minutes for a polished live demo, or 60 to 90 seconds for a very tight showcase. |
| Tone | Smart, polished, slightly playful, and credible without sounding overhyped. |

## Key talking points

| # | Point |
| --- | --- |
| 1 | The system improves decisions by asking targeted questions rather than giving generic advice. |
| 2 | It preserves provenance: every insight should be traceable to evidence. |
| 3 | It distinguishes between verified, contradicted, and unknown. |
| 4 | It keeps the human in control and makes terms explainable. |

## Visual flow

| Step | What to show |
| --- | --- |
| 1 | Open with a simple title card: “VibeCheck — evidence-backed decision making” |
| 2 | Show a mock interviewer interface or transcript panel |
| 3 | Display a few structured outputs: hypotheses, evidence cards, contradictions, and a term sheet |
| 4 | End with a clean summary of the product’s value proposition |
